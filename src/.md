## 元胞自动机开发
 * 康威生命游戏
 *
 * 生命游戏的规则非常简单：在一个二维网格上，每个格子代表一个细胞，细胞可以是 alive（活的）或者 dead（死的）。每一代，所有细胞会同时根据以下规则更新状态：
 * 1. 如果一个 alive 的细胞周围有 2 或 3 个 alive 的邻居，它在下一代继续 alive；否则它会变成 dead。
 * 2. 如果一个 dead 的细胞周围恰好有 3 个 alive 的邻居，它在下一代变成 alive；否则它继续 dead。
 *
 * 生命棋盘的演化逻辑很简单，就是两个棋盘，一个current，一个next，current记录当前状态，next根据current计算下一代状态。
 * 每一代演化完成后，next就变成current，继续下一轮演化。

### 架构演进以及逐步优化的过程
根据生命游戏的规则，我们写出一个单元格的基本状态代码。

```js
let current_cell = 'alive';
let next_cell =  'dead';

let nbors = 0;

switch (current_cell) {
  case "dead":
    {
      if (nbors === 3) next_cell = "alive";
      else next_cell = "dead";
    }
    break;
  case "alive":
    {
      if (nbors === 2 || nbors === 3) next_cell = "alive";
      else next_cell = "dead";
    }
    break;
}
````

实际上我们可以制造一个状态机表格：
```js
const GOL = {
  'dead': {
    '3': 'alive',
    'otherwise': 'dead'
  },
  'alive': {
    '2': 'alive',
    '3': 'alive',
    'otherwise': 'dead'
  }
}
```
但是js对象并没有一个很好的map机制，也就是otherwise没有办法自动被匹配到。那么我们可以考虑做成数组：
```js
// 由于我们的二维画布里面，一个cell周围最多八个邻居，所以数组长度为9即可。
const GOL = {
  'dead': ['dead', 'dead', 'dead', 'alive', 'dead', 'dead', 'dead', 'dead', 'dead'],
  'alive': ['dead', 'dead', 'alive', 'alive', 'dead', 'dead', 'dead', 'dead', 'dead'],
}
```

```js
 * T = [
 *    dead table (五个 dead 三个 alive，此时这个dead的细胞才能活过来)

 ```
